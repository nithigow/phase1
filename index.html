Step 1 User fills intake form

This is the entry point of your entire system. The user opens your web UI or hits your API and submits all the information needed to start a project.

  • Project name — e.g. "Health Insurance Portal". Backend converts it to a slug like health-insurance-portal.
  • App URL — the website to test. e.g. https://staging.healthportal.com. Playwright will open this later.
  • Environment — dropdown: staging, dev, or production.
  • Files — user uploads brd.pdf, fsd.pdf, etc.
  • Auth type — JWT, session cookie, or OAuth.
  • API endpoints — Swagger URL or manually entered rows.
Example — credentials table
Role: admin | Email: admin@test.com | Password: Admin@123
Role: user | Email: user@test.com | Password: User@123
Role: guest | Email: guest@test.com | Password: Guest@123

FastAPI receives all of this as a single POST request and passes it down the pipeline.

Step 2 Validate inputs

Catch mistakes before touching Supabase at all. If any check fails, return an error and stop — no rows inserted, no files uploaded.

  • Project name present? If blank → "Project name is required".
  • App URL valid? Must start with http:// or https://.
  • At least one file uploaded? Minimum one BRD required.
  • File format allowed? Only .pdf and .csv. Reject .docx, .xlsx etc.
  • At least one credential set? Phase 3 Playwright sessions need something to log in with.
Step 3 Create project record  Supabase Postgres

Insert a row into the Supabase projects table. This is the source of truth — every later step updates this row, and Phase 2 agents query it.

Python — supabase-py
import uuid, os
from supabase import create_client

supabase = create_client(
    os.environ["SUPABASE_URL"],
    os.environ["SUPABASE_KEY"]
)

project_id = str(uuid.uuid4())
# "a3f2c1d4-9e87-4b2a-bc13-5f7d9e2a1c88"

supabase.table("projects").insert({
    "project_id":   project_id,
    "project_name": "Health Insurance Portal",
    "app_url":      "https://staging.healthportal.com",
    "environment":  "staging",
    "auth_type":    "JWT",
    "status":       "ingestion_in_progress"
}).execute()

Step 4 URL health check

Send a GET request to the app URL. If it fails, update the Supabase projects row status and halt — no point processing files for an unreachable app.

Python
import httpx

try:
    response = httpx.get(app_url, timeout=10)
    if response.status_code >= 400:
        supabase.table("projects").update(
            {"status": "url_unreachable"}
        ).eq("project_id", project_id).execute()
        return error("App returned status " + str(response.status_code))
except httpx.TimeoutException:
    supabase.table("projects").update(
        {"status": "url_unreachable"}
    ).eq("project_id", project_id).execute()
    return error("App did not respond within 10 seconds")
Real example
Staging server was shut down over the weekend. Health check times out. Supabase projects row updated to url_unreachable. User sees: "The app is not responding. Please check the URL."

On success, optionally store the response headers (e.g. X-Powered-By: Express) in the Supabase row — useful tech stack context for later agents.

Step 6 Detect file type

Download the file from Supabase Storage into memory, then run detection. Two questions: is it PDF or CSV? And if PDF, is it text-based or scanned?

Python — download from Supabase then detect
import fitz
from io import BytesIO

# Download from Supabase Storage into memory — no local disk needed
file_bytes = supabase.storage.from_("documents").download(raw_path)

def is_scanned(pdf_bytes):
    doc = fitz.open(stream=pdf_bytes, filetype="pdf")
    total_text = "".join(page.get_text() for page in doc)
    return len(doc) > 2 and len(total_text.strip()) < 50
Scanned example
15-page insurance policy. PyMuPDF extracts 3 chars total → flagged scanned → Vision LLM path.
Text-based example
PDF exported from Word. Thousands of characters extracted immediately → normal PyMuPDF path.
Step 7 Extract raw text

Three sub-paths — all operating on bytes in memory (already downloaded from Supabase in Step 6). All three produce the same output: one plain text string.

Path A — Text PDF via PyMuPDF:

Python
def extract_pdf(pdf_bytes):
    doc = fitz.open(stream=pdf_bytes, filetype="pdf")
    return "".join(
        f"\n--- Page {i+1} ---\n" + page.get_text()
        for i, page in enumerate(doc)
    )

Path B — Scanned PDF via Vision LLM:

Python
from pdf2image import convert_from_bytes
import anthropic, base64

client = anthropic.Anthropic()

def extract_scanned(pdf_bytes):
    images = convert_from_bytes(pdf_bytes, dpi=200)
    full_text = ""
    for img in images:
        buf = BytesIO()
        img.save(buf, format="PNG")
        b64 = base64.b64encode(buf.getvalue()).decode()
        resp = client.messages.create(
            model="claude-opus-4-5", max_tokens=4096,
            messages=[{"role": "user", "content": [
                {"type": "image", "source": {"type": "base64",
                    "media_type": "image/png", "data": b64}},
                {"type": "text", "text": "Extract all text from this page exactly."}
            ]}]
        )
        full_text += resp.content[0].text
    return full_text

Path C — CSV via pandas:

Python
import pandas as pd

def extract_csv(csv_bytes):
    df = pd.read_csv(BytesIO(csv_bytes))
    return df.to_string(index=False)

Step 10 Store credentials  Supabase Postgres

Encrypt credentials with Fernet, then insert into the Supabase credentials table. Never store a plain text password anywhere — not in Supabase, not in logs, nowhere.

Python — Fernet encrypt → Supabase Postgres
from cryptography.fernet import Fernet

# Key lives in env variable — NEVER hardcode
fernet = Fernet(os.environ["CREDENTIAL_ENCRYPTION_KEY"])

def encrypt(plain): return fernet.encrypt(plain.encode()).decode()
def decrypt(enc):   return fernet.decrypt(enc.encode()).decode()

for cred in credentials:
    supabase.table("credentials").insert({
        "project_id":         project_id,
        "role_name":          cred["role"],
        "username":           cred["email"],
        "encrypted_password": encrypt(cred["password"]),
        "login_url":          cred.get("login_url", app_url + "/login"),
        "totp_secret":        encrypt(cred["totp_secret"]) if cred.get("totp_secret") else None
    }).execute()
Row in Supabase credentials table
role_name: admin
username: admin@test.com
password: gAAAAABl7k2X9mPqR...  (encrypted — useless without the key)
login_url: https://staging.healthportal.com/admin/login

When Phase 3 needs to log in, it queries Supabase, calls decrypt() at runtime, uses the password for Playwright, then immediately discards it. Decrypted password never touches disk or any log.
